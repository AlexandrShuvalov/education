# ОГЭ Навигатор

Стартовый прототип сайта для подготовки к ОГЭ по математике. Проект сделан как статический сайт без сборки и открывается двойным кликом по `index.html`.

## Что уже есть

- Главная страница с позиционированием продукта.
- Один предмет: математика.
- Выбор номера задания ОГЭ от 1 до 25.
- Полноэкранный просмотр всех прототипов выбранного номера.
- Выбор нужных прототипов галочками.
- Сохранение выбранных прототипов при переключении между номерами.
- Удаление отдельных выбранных прототипов из будущего листа.
- Блок пробных вариантов.
- Заготовка сценариев для кабинета учителя.
- Конструктор печатных листов: выбранные прототипы можно сохранить в PDF или распечатать.
- Отдельные файлы базы по номерам ОГЭ: `tasks/task-01.js`, `tasks/task-02.js`, ..., `tasks/task-25.js`.

## Как открыть

Откройте файл `index.html` двойным кликом в браузере.

Локальный сервер больше не обязателен. Если он все же нужен:

```powershell
py -m http.server 8000
```

После запуска откройте в браузере `http://localhost:8000`.

## Как добавлять задачи

### Самый простой способ: через редактор

Откройте `task-editor/index.html` двойным кликом. Это отдельный локальный сайт-редактор, он не подключается к рабочему сайту.

В нем можно:

- выбрать ОГЭ или ЕГЭ;
- выбрать номер задания;
- заполнить условие, формулу, варианты ответа, картинку, ответ и решение;
- сразу посмотреть предпросмотр задания;
- добавить несколько заданий за одну сессию;
- скачать готовый файл `task-XX.js`.

После скачивания замените старый файл в папке `tasks` или `ege-tasks` новым файлом с тем же именем.
Например, если редактор скачал `task-06.js`, положите его вместо `tasks/task-06.js`.

Для картинок сначала положите изображение в папку вида `tasks/images/task-07/`, а в редакторе укажите путь
`images/task-07/название-файла.png`.

### Ручной способ

Все прототипы разнесены по файлам в папке `tasks`. Например, прототипы задания 6 хранятся в `tasks/task-06.js`, прототипы задания 15 нужно добавлять в `tasks/task-15.js`.

Каждый файл выглядит так:

```js
window.OGE_TASKS = window.OGE_TASKS || [];
window.OGE_TASKS.push(
  // сюда добавляются прототипы этого номера
);
```

Чтобы добавить новый прототип, откройте файл нужного номера, вставьте объект внутрь `window.OGE_TASKS.push(...)`, поменяйте данные и поставьте запятую между задачами.

Пример одной задачи:

```json
{
  "id": "math-oge-006-p09",
  "subject": "Математика",
  "number": 6,
  "prototype": 9,
  "title": "Дроби и степени",
  "topic": "Действия с десятичными дробями",
  "block": "",
  "prompt": "Найдите значение выражения",
  "section": "Алгебра",
  "difficulty": "easy",
  "question": "Найдите значение выражения: 12 - 4,5.",
  "answer": "7,5",
  "solution": "12 - 4,5 = 7,5."
}
```

Поле `number` — это номер задания ОГЭ от 1 до 25. Поле `prototype` — номер прототипа внутри этого задания. Для заданий 1-19 сайт показывает тип `Краткий ответ`, для заданий 20-25 — `Полное решение`.

Для заданий 1-19 особенно важно заполнить `answer`. Для заданий 20-25 особенно важно заполнить `solution`, потому что там требуется полное решение.

Поле `title` идет в крупный заголовок вида `О6. Дроби и степени`. Поле `block` показывает строку под заголовком, например ``. Поле `prompt` задает формулировку после `Задание 1.`, например `Найдите значение выражения`.

## Что делать дальше

1. Наполнить файлы в папке `tasks` реальными прототипами по математике.
2. Добавить группировку прототипов по темам внутри каждого номера ОГЭ.
3. Сделать хранение прогресса в `localStorage`.
4. Добавить генератор вариантов по темам и уровню сложности.
5. Позже подключить backend, регистрацию, кабинет ученика и кабинет учителя.

## Как печатать и сохранять PDF

1. Откройте раздел `Номера ОГЭ`.
2. Выберите номер задания от 1 до 25.
3. В полноэкранном окне отметьте галочками нужные прототипы.
4. Откройте раздел `PDF и печать`.
5. При необходимости удалите отдельные выбранные прототипы кнопкой `Удалить`.
6. При необходимости включите `Добавить ответы и решения`.
7. Нажмите `Сохранить в PDF`, чтобы открыть отдельное окно с готовым листом.
8. В открывшемся окне нажмите `Скачать PDF`, чтобы сохранить лист PDF-файлом.
9. Нажмите `Распечатать`, если нужно отправить лист на принтер или сохранить через печать браузера.

Кнопка `Сохранить в PDF` открывает отдельное окно с красивой версткой заданий. В этом окне есть отдельные кнопки `Скачать PDF` и `Распечатать`. Если браузер заблокирует новое окно, разрешите всплывающие окна для этого файла. В режиме печати сайт скрывает меню, главную страницу и интерактивные блоки. На листе остается только подготовленная работа.

## Важное замечание

Не стоит копировать дизайн, тексты и базу заданий чужих сайтов. Лучше использовать похожую образовательную логику, но собственный интерфейс, структуру материалов и авторские задания.
# Личный кабинет репетитора и ученика

Страница `cabinet.html` открывает демонстрационную версию кабинета. Она работает без сервера:
данные сохраняются только в `localStorage` текущего браузера. Такой режим нужен для проверки
интерфейса, но не подходит для настоящих учеников и реальной переписки.

В кабинете уже можно:

- добавить ученика и получить отдельный код входа;
- назначить домашнюю работу конкретному ученику;
- добавить ученика в расписание занятий;
- открыть форму занятия кликом по свободному слоту календаря и переносить занятия мышкой;
- добавить общий или персональный материал по теме;
- написать сообщение от лица репетитора или ученика;
- открыть ученический кабинет по демо-коду `NE-IVAN-25` или `NE-MARIA-80`.

Для онлайн-версии с реальными аккаунтами подготовлен файл `supabase-schema.sql`. Он создаёт
таблицы профилей, связей репетитор-ученик, занятий, домашних работ, материалов и сообщений, а также
включает Row Level Security: ученик видит только свои данные, репетитор только своих учеников.

## Подключение Supabase

1. Создайте проект на `https://supabase.com`.
2. Откройте SQL Editor и выполните содержимое файла `supabase-schema.sql`.
3. Включите вход пользователей по email в разделе Authentication.
4. После этого замените демонстрационный слой хранения в `cabinet.js` на подключение Supabase.

Важно: демонстрационные коды входа из `cabinet.js` нельзя использовать как пароль в
онлайн-версии. Для реального запуска учеников нужно приглашать по email через Supabase Auth.
